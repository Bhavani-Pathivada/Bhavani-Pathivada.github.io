The Storage World

What I work with every day

SAN — Storage Area Network

A dedicated high-speed network that connects servers to storage devices using Fibre Channel or iSCSI protocols. It presents storage as locally attached block devices to hosts — critical for databases, VMs, and latency-sensitive workloads. Zoning controls which hosts can see which storage ports.

NAS — Network-Attached Storage

File-level storage accessed over standard network protocols like NFS (Linux) and CIFS/SMB (Windows). NetApp ONTAP excels here — offering unified storage that serves both SAN and NAS from the same controllers, with rich data management features like SnapMirror replication.

IBM SVC / Storwize

IBM's SAN Volume Controller is a powerful virtualization layer that sits between servers and back-end storage. It enables thin provisioning, Easy Tier (automated tiering), FlashCopy snapshots, and Metro/Global Mirror replication — providing abstraction that makes storage hardware-agnostic to hosts.

HPE 3PAR / Alletra

HPE's flagship all-flash arrays built around a full mesh ASIC architecture — every node connects to every other, eliminating bottlenecks. Alletra is the cloud-native evolution: consumption-based, API-first, and integrated with HPE GreenLake for hybrid cloud storage management.
